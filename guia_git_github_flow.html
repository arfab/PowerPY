Cheat Sheet Local

PowerPY — Guía Git & GitHub Flow

Manual de referencia rápido para mantener tu código seguro en la nube y trabajar de forma profesional usando GitHub Flow.

✓

¡Proyecto Subido Exitosamente!

Tu proyecto ya está alojado de forma segura en https://github.com/arfab/PowerPY. A modo de referencia histórica, aquí están los pasos que ejecutamos para subirlo por primera vez:

1

Inicializar Repositorio Local

Crea la base de datos de historial de Git en tu carpeta local.

PowerShell 
git init
💡
Nota: En este paso, Git comienza a trackear tu proyecto y respeta las reglas configuradas en el archivo .gitignore para omitir archivos innecesarios o confidenciales.
2

Guardar Cambios Localmente (Commit)

Agrega todos los archivos válidos al "Stage" y confirma tu primer punto de restauración con un commit descriptivo.

PowerShell 
git add .
git commit -m "Initial commit: Estructura base del proyecto PowerPY"
3

Configurar Rama y Vincular Servidor

Renombra la rama principal a main y establece la conexión con tu repositorio remoto de GitHub.

PowerShell 
# Cambiar nombre de rama predeterminada a main
git branch -M main

# Vincular con GitHub
git remote add origin https://github.com/arfab/PowerPY.git
4

Subir Código a la Nube

Envía la rama main de tu computadora local a GitHub.

PowerShell 
git push -u origin main
!

Regla de Oro de GitHub Flow

NUNCA programes directamente sobre la rama main. Todo cambio nuevo debe nacer en una rama secundaria, ser revisado en un Pull Request y luego fusionarse de vuelta en main.

1

Crear una Rama Nueva

Asegúrate de tener la última versión de main y crea una rama descriptiva.

PowerShell 
# Ir a main
git checkout main

# Descargar últimas actualizaciones
git pull origin main

# Crear y cambiarse a la nueva rama
git checkout -b feature/nombre-de-tu-mejora
2

Hacer Cambios Locales

Escribe tu código. Haz commits breves y atómicos conforme resuelvas cada parte.

PowerShell 
# Ver qué archivos modificaste
git status

# Preparar y guardar cambios
git add .
git commit -m "Implementa funcionalidad X en app.py"
3

Subir la Rama a GitHub

Sube tu rama al servidor remoto para poder visualizarla en GitHub.

PowerShell 
git push -u origin feature/nombre-de-tu-mejora
4

Crear un Pull Request

Ve a tu GitHub, haz clic en el banner amarillo "Compare & Pull Request", detalla lo que has hecho y crea la solicitud de fusión.

⚡
Esto permite revisar el código antes de integrarlo permanentemente.
5

Merge (Fusión) y Limpieza

Una vez aprobado en la web, haz clic en **"Merge Pull Request"** en GitHub. Luego, limpia tus ramas en local:

PowerShell 
# Vuelve a tu main local
git checkout main

# Descarga los cambios fusionados desde GitHub
git pull origin main

# Elimina la rama local que ya no necesitas
git branch -d feature/nombre-de-tu-mejora
⚠️

Error: "Remote origin already exists"

Este es el error que te ocurrió al intentar definir de nuevo tu servidor. Sucede porque la variable origin ya está asignada a otra URL.

Solución Directa (Modificar):

PowerShell 
git remote set-url origin https://github.com/arfab/PowerPY.git

Solución Alternativa (Borrar y Agregar):

PowerShell 
git remote remove origin
git remote add origin https://github.com/arfab/PowerPY.git
💡

Cómo revertir un commit incorrecto localmente

Si hiciste un commit pero olvidaste agregar algo o te equivocaste en el mensaje del commit:

PowerShell 
# Deshacer el último commit guardando tus archivos modificados intactos
git reset --soft HEAD~1
💡

Verificar el estado de tus remotos

Para asegurarte de a dónde se enviará tu código con exactitud:

PowerShell 
git remote -v
